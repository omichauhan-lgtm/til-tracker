## Category: Database Transactions
### **Optimistic vs Pessimistic Concurrency Control**

Pessimistic locking prevents conflicts by acquiring shared or exclusive locks on data before modification, risking deadlocks. Optimistic locking assumes conflicts are rare, checking if data has been modified by another transaction (via version numbers or timestamps) before committing, retrying if a conflict is detected.
