### **Idempotency Keys in Payment & API Gateway Design**

To prevent duplicate transactions, client requests include a unique idempotency key. The server stores this key along with the response in a cache (e.g. Redis) with a TTL. If a retry with the same key is received, the server returns the cached response directly instead of re-executing the operation.
